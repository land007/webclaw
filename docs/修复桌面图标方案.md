# 修复桌面图标不显示问题

## 问题背景

Docker 容器中的 GNOME Flashback 桌面不显示图标。根本原因是 `configs/xsession` 脚本手动启动了各个会话组件（metacity、gsd-*、gnome-panel），但 **从未启动 `gnome-flashback`**——这是负责绘制桌面窗口和渲染 `~/Desktop/*.desktop` 图标文件的组件。

在 Ubuntu 24.04 的现代 GNOME Flashback 中，Nautilus 已不再处理桌面图标（从 GNOME 3.28 起移除）。`gnome-flashback` 二进制程序同时负责桌面背景和桌面图标的显示。

此外，当前壁纸通过 `hsetroot` 设置（直接写入 X11 根窗口）。当 `gnome-flashback` 启动后，它会创建自己的桌面窗口覆盖在根窗口之上，导致 `hsetroot` 设置的壁纸不可见。壁纸需要改用 `gsettings` 设置，这样 `gnome-flashback` 才能正确渲染。

## 根本原因分析

正常的 GNOME Flashback 会话（`gnome-session-flashback`）启动时，`gnome-flashback-metacity.session` 文件定义的 RequiredComponents 包含：
- `gnome-flashback-init` — 初始化辅助程序
- `gnome-flashback` — 核心服务（桌面图标、背景等）
- `gnome-panel` — 面板/任务栏
- `metacity` — 窗口管理器
- `gnome-settings-daemon` — 设置守护进程

当前 `xsession` 只启动了 metacity、gsd-xsettings、gsd-keyboard、gnome-panel，缺少了关键的 `gnome-flashback` 组件。

## 修改方案

### 1. `configs/xsession` — 启动 gnome-flashback + 壁纸改用 gsettings

**壁纸部分**（第 73-83 行）：将 `hsetroot -cover` 替换为 `gsettings`，保留 hsetroot 作为降级方案。

修改前：
```bash
if [ "$INITIAL_THEME" = "light" ]; then
    WP="${LIGHT_WALLPAPERS[$((RANDOM % ${#LIGHT_WALLPAPERS[@]}))]}"
    hsetroot -cover "$WP" 2>/dev/null || xsetroot -solid '#f0f0f0'
    xsetroot -cursor_name left_ptr
    export GTK_THEME=Adwaita
else
    WP="${DARK_WALLPAPERS[$((RANDOM % ${#DARK_WALLPAPERS[@]}))]}"
    hsetroot -cover "$WP" 2>/dev/null || xsetroot -solid '#0d1117'
    xsetroot -cursor_name left_ptr
    export GTK_THEME=Adwaita-dark
fi
```

修改后：
```bash
if [ "$INITIAL_THEME" = "light" ]; then
    WP="${LIGHT_WALLPAPERS[$((RANDOM % ${#LIGHT_WALLPAPERS[@]}))]}"
    export GTK_THEME=Adwaita
else
    WP="${DARK_WALLPAPERS[$((RANDOM % ${#DARK_WALLPAPERS[@]}))]}"
    export GTK_THEME=Adwaita-dark
fi

# 设置壁纸：优先使用 gsettings（gnome-flashback 桌面使用），
# 降级使用 hsetroot（直接绘制根窗口）。
GF_BIN=$(command -v gnome-flashback 2>/dev/null || true)
[ -z "$GF_BIN" ] && [ -x /usr/libexec/gnome-flashback ] && GF_BIN=/usr/libexec/gnome-flashback
if [ -n "$GF_BIN" ]; then
    gsettings set org.gnome.desktop.background picture-uri "file://$WP" 2>/dev/null
    gsettings set org.gnome.desktop.background picture-uri-dark "file://$WP" 2>/dev/null
else
    hsetroot -cover "$WP" 2>/dev/null || xsetroot -solid '#0d1117'
    xsetroot -cursor_name left_ptr
fi
```

**设置守护进程之后**（第 93 行之后，gnome-panel 之前）：插入 `gnome-flashback` 启动：

```bash
# ─── GNOME Flashback 桌面（绘制背景 + 桌面图标）─────
GF_BIN=$(command -v gnome-flashback 2>/dev/null || true)
[ -z "$GF_BIN" ] && [ -x /usr/libexec/gnome-flashback ] && GF_BIN=/usr/libexec/gnome-flashback
if [ -n "$GF_BIN" ]; then
    "$GF_BIN" &
    sleep 0.5
fi
```

注意：`gnome-flashback` 二进制文件在 Ubuntu 24.04 上可能位于 `/usr/libexec/gnome-flashback`（不在 PATH 中），需要做路径搜索。

### 2. `scripts/theme-switch.sh` — 壁纸方式从 hsetroot 改为 gsettings

在壁纸数组定义之后（约第 33 行）添加辅助函数：

```bash
# 设置壁纸：检测 gnome-flashback 是否运行，选择合适的方式。
set_wallpaper() {
    local wp="$1"
    if pgrep -x gnome-flashback >/dev/null 2>&1; then
        gsettings set org.gnome.desktop.background picture-uri "file://$wp" 2>/dev/null
        gsettings set org.gnome.desktop.background picture-uri-dark "file://$wp" 2>/dev/null
    else
        hsetroot -cover "$wp" 2>/dev/null
    fi
}
```

替换两处 `hsetroot -cover "$WALLPAPER"` 调用（第 38 行和第 52 行）为 `set_wallpaper "$WALLPAPER"`。

### 3. `Dockerfile` — 添加 gnome-flashback 的 dconf 默认值

在步骤 6b（dconf 配置块，第 38-48 行）中，在 `dconf update` 之前添加两个新的 dconf 键文件：

```bash
        && printf '[org/gnome/gnome-flashback/desktop]\nshow-icons=true\n' \
           > /etc/dconf/db/local.d/01-flashback-desktop-icons \
        && printf '[org/gnome/desktop/background]\npicture-uri='"'"''"'"'\npicture-uri-dark='"'"''"'"'\nprimary-color='"'"'#0d1117'"'"'\n' \
           > /etc/dconf/db/local.d/02-background-defaults \
```

### 4. `scripts/startup.sh` — 确保 show-icons 在 Docker 卷持久化后不丢失

在现有的 `dconf reset -f /org/gnome/gnome-panel/`（第 312 行）之后添加：

```bash
    # 确保桌面图标已启用（处理 Docker 卷中的过期 dconf 数据）
    sudo -u ubuntu dbus-run-session -- dconf write /org/gnome/gnome-flashback/desktop/show-icons true 2>/dev/null || true
```

## 组件启动顺序

修改后的完整启动顺序：
1. metacity（窗口管理器）
2. gsd-xsettings、gsd-keyboard（设置守护进程）
3. **gnome-flashback（桌面图标 + 背景）** ← 新增
4. gnome-panel（任务栏）
5. theme-switch（主题同步）
6. fcitx（输入法）

## 验证方法

1. 重新构建 Docker 镜像，以桌面模式启动容器
2. 通过 noVNC 连接 — 确认桌面图标显示（Chrome、VS Code、Vibe Kanban 等）
3. 确认壁纸正确显示
4. 运行 `theme-switch light` / `theme-switch dark` — 确认壁纸切换正常
5. 重启容器 — 确认图标和壁纸保持不变
6. 在终端执行 `pgrep -a gnome-flashback` 确认进程在运行
7. 在终端执行 `gsettings get org.gnome.gnome-flashback.desktop show-icons` 确认返回 `true`
